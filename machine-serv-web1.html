MachineSERV-WEB1

ÉTAPE 1 : Création de la machine virtuelle

  1. Sur VMware, depuis l'interface principale, cliquez surCreate a New Virtual Machine

  2. SélectionnezTypical (recommended)puis cliquez surNext >

  3. SélectionnezI will install the operating system later.puis cliquez surNext >

  4. SélectionnezLinux, et sélectionnez la VersionDebian 10.x 64-bit,puis cliquez surNext >

  5. Saisissez le nom de la machine virtuelle en-dessus deVirtual machine name:,puis cliquez surNext >

  6. SélectionnezSplit virtual disk into multiple files,puis cliquez surNext >

  7. Cliquez surFinish

  8. Cliquez surEdit virtual machine settings

  9. Cliquez surCD/DVD (IDE)

  10. SélectionnezUse ISO image file:, puis cliquez surBrowse...pour rechercher l'iso de Debian téléchargé.

  11. Cliquez surOKpour valider et fermer la fenêtre.

ÉTAPE 2 : Installation de Debian

  1. Cliquez surPower on this virtual machine

  2. Cliquez sur l'écran et sélectionnezInstallen appuyant sur la touche Entrée du clavier.

  3. SélectionnezFrench - Françaiscomme langage pour l'installation.

  4. SélectionnezFrancecomme choix de situation géographique.

  5. SélectionnezFrançaiscomme disposition de clavier à utiliser.

  6. Saisissez le nom de la machine :SERV-WEB1

  7. Saisissez le mot de passe du superutilisateur :root

  8. Confirmez le mot de passe.

  9. Saisissez votre prénom comme nouvel utilisateur.

  10. Saisissez de nouveau votre prénom pour l'identifiant du compte utilisateur. Normalement, il est déjà saisi.

  11. Saisissez un mot de passe pour ce compte. Mot de passe conseillé :root(très sécurisé)

  12. Confirmez ce dernier.

  13. SélectionnezAssisté - utiliser un disque dur entier

  14. SélectionnezSCSI3 (0,0,0) (sda) - 21.5 GB VMware, VMware Virtual S

  15. SélectionnezTout dans une seule partition (recommandé pour les débutants)

  16. SélectionnezTerminer le partitionnement et appliquer les changements

  17. Faut-il appliquer les changements sur les disques ?-><Oui>

  18. Faut-il analyser d'autres supports d'installation ?-><Non>

  19. SélectionnezFrancecomme pays du miroir de l'archive Debian.

  20. Sélectionnezdeb.debian.orgcomme miroir de l'archive Debian.

  21. Souhaitez-vous participer à l'étude statistique sur l'utilisation des paquets ?-><Non>

  22. Décochez tous à l'aide de la touche Espace, puis la touche Entrée pour<Continuer>

  23. Installer le programme de démarrage GRUB sur le disque principal ?-><Oui>

  24. Sélectionnez/dev/sdacomme périphérique où sera installé le programme de démarrage GRUB.

  25. Sélectionnez<Continuer>pour terminer l'installation.

Debian est maintenant installé sur la machineSERV-WEB1!

ÉTAPE 3 : Mise en place du Cluster Web

Tips #1

Vous pouvez faire un netoyage de la console à l'aide de la commandeclear

  1. Lancez la machineSERV-WEB1(si ce n'ai pas fait)

  2. SélectionnezDebian GNU/Linux

  3. SERV-WEB1 login:->root

    Password:->root

  4. Installez apache2, php, wget et unzip à l'aide de la commandeapt

    root@SERV-WEB1:~# 
    apt-get update
    root@SERV-WEB1:~# 
    apt install -y apache2 php wget unzip

    À quoi sert la commandeapt-get update?

    apt-getpermet d'effectuer l'installation et la désinstallation de paquets en provenance d'un dépôt APT. updatepermet de mettre à jour la liste des fichiers disponibles dans les dépôts APT présents dans le fichier de configuration/etc/apt/sources.list.

    Pourquoi faire unapt-get update?

    Permet d'avoir aucune erreur lors d'une installation.

    À quoi sert le-ydans la commandeapt install -y apache2 php wget unzip?

    -ypermet de répondre Oui (O) par défaut aux questions qui sont posés pendant l'installation.

    Et si une question importante attendait Non (N) comme réponse ?

    (Réponse en cours de rédaction...)

  5. Accédez au dossier racine par défaut d'apache2.

    root@SERV-WEB1:~# 
    cd /var/www/html

  6. Téléchargez le templatethegrillà l'aide de la commandewget

    root@SERV-WEB1:/var/www/html# 
    wget https://github.com/technext/thegrill/archive/master.zip

  7. Décompressez l'archivemaster.zipà l'aide de la commandeunzip

    root@SERV-WEB1:/var/www/html# 
    unzip master.zip

  8. Changez le dossier racine par défaut d'apache2 par celui dethegrill-masterdans le fichier000-default.conf

    Pour ce faire, ouvrez le fichier000-default.confà l'aide de la commandenano

    Le fichier000-default.confse trouve dans le répertoire/etc/apache2/sites-available/

    Dans le fichier, rajoutez/thegrill-masteraprès/var/www/html->DocumentRoot /var/www/html/thegril-master

    root@SERV-WEB1:/var/www/html# 
    nano /etc/apache2/sites-available/000-default.conf
    Enregistrez la modification avecctrl x->O (la lettre)->Entrée

  9. Redémarrez le service d'apache2 à l'aide de la commanderestart

    root@SERV-WEB1:/var/www/html# 
    service apache2 restart

  10. Dans le fichierthegrill-master/index.html, ajoutez respectivement 1 après le titreThe Grill

    root@SERV-WEB1:/var/www/html# 
    nano thegrill-master/index.html
    Enregistrez la modification avecctrl x->O (la lettre)->Entrée

  11. Quittez le répertoire/var/www/htmlà l'aide de la commandecd

    root@SERV-WEB1:/var/www/html# 
    cd

  12. Modifiez la carte réseau de la machine.

    Pourquoi modifier la carte réseau de la machine ?

    La machine virtuelleSERV-WEB1est par défaut en NAT.
    Le NAT est utilisé pour partager l'adresse IP de l'hôte.
    Cette adresse IP est généré automatiquement sur l'interfaceens33. Vous pouvez vérifier à l'aide de la commandeip a.
    Dans la prochaine étape, vous allez devoir attribuer une adresse IP static à cette machine virtuelle.
    Et pour attribuer une adresse IP static à une lachine, il faut qu'elle appartienne à un segment LAN.
    Il va donc faloir créer ce segement LAN dans les paramères de la machine virtuelle.
    1. Faites un clique droit sur le nom de la machine en haut, et cliquez surSettings...
    2. Une fois dans les paramètres de la machine virtuelle, cliquez surNetwork Adapter
    3. À droite, cliquez surLAN segment:en-bas.
    4. Plus bas, cliquez surLAN Segments...pour créer un segment LAN.
    5. Cliquez surAdd
    6. Saisissez le nom du segement :LAN
    7. Cliquez surOKpour valider et créer ce nouveau segement.
    8. En-dessous deLAN segment:, cliquez et sélectionnez le segement nomméLAN
    9. Une fois fait, cliquez surOKen-bas à droite pour fermer la fenêtre des paramètres de la machine virtuelle.

  13. Modifiez le fichierinterfacessitué dans le répertoire/etc/network/

    Remplacezdhcpparstatic

    Et rajoutez en-dessous l'adresse IP de la machine virtuelleaddress 172.20.0.21/24

    root@SERV-WEB1:~# 
    nano /etc/network/interfaces
    Enregistrez la modification avecctrl x->O (la lettre)->Entrée

  14. Redémarrez la machine virtuelle à l'aide de la commandereboot(et se reconnecter).

    root@SERV-WEB1:~# 
    reboot

  15. Vérifiez la configuration IP de la machine virtuelle à l'aide des commandesip aetifconfig

    root@SERV-WEB1:~# 
    ip a
    root@SERV-WEB1:~# 
    ifconfig

    Success!

    L'adresse IP de la machineSERV-WEB1est bien172.20.0.21

ÉTAPE 4 : Clonage de la machine virtuelle

  1. Eteignez la machine virtuelleSERV-WEB1en cliquant sur la petite croix en haut à droite du nom de la machine.

    Puis cliquez surPower Offpour éteindre la machine.

  2. À gauche, cliquez sur le nom de la machine virtuelleSERV-WEB1

  3. Une fois cliquer, faites un clique droit dessus et allez dansManage, et cliquez surClone...

  4. Cliquez surSuivant >

  5. SélectionnezThe current state in the virtual machine, puis cliquez surSuivant >

  6. SélectionnezCreate a full clone, puis cliquez surSuivant >

  7. Saisissez le nom de la nouvelle machine virtuelle :SERV-WEB2, puis cliquez surTerminer

  8. Cliquez surClose
